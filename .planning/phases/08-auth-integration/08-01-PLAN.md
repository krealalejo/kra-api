    ```java
    package com.kra.api.infrastructure.config;

    import org.springframework.context.annotation.Bean;
    import org.springframework.context.annotation.Configuration;
    import org.springframework.http.HttpMethod;
    import org.springframework.security.config.annotation.web.builders.HttpSecurity;
    import org.springframework.security.config.annotation.web.configuration.EnableWebSecurity;
    import org.springframework.security.web.SecurityFilterChain;
    import static org.springframework.security.config.Customizer.withDefaults;

    /**
     * Spring Security configuration for OAuth2 Resource Server with AWS Cognito.
     *
     * - GET /projects/**:                  public (no JWT required)
     * - POST /projects/**:                 requires valid Cognito JWT
     * - PUT /projects/**:                  requires valid Cognito JWT
     * - DELETE /projects/**:               requires valid Cognito JWT
     * - Any other request:                 requires authentication (safe default)
     *
     * JWT validation is configured via:
     *   spring.security.oauth2.resourceserver.jwt.issuer-uri=https://cognito-idp.eu-west-1.amazonaws.com/eu-west-1_sDg68GJKt
     *
     * Testing:
     *   Use @Import(SecurityConfig.class) in @WebMvcTest test classes.
     *   Use .with(jwt()) MockMvc post-processor for authenticated requests.
     */
    @Configuration
    @EnableWebSecurity
    public class SecurityConfig {

        @Bean
        public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
            http
                .authorizeHttpRequests((authorize) -> authorize
                    // GET requests: public, no authentication required
                    .requestMatchers(HttpMethod.GET, "/projects/**").permitAll()
                    // Write operations: require valid OAuth2 JWT
                    .requestMatchers(HttpMethod.POST, "/projects/**").authenticated()
                    .requestMatchers(HttpMethod.PUT, "/projects/**").authenticated()
                    .requestMatchers(HttpMethod.DELETE, "/projects/**").authenticated()
                    // Any other requests: authenticated (safe default)
                    .anyRequest().authenticated()
                )
                .oauth2ResourceServer((oauth2) -> oauth2
                    .jwt(withDefaults())
                );

            return http.build();
        }
    }
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST/PUT/DELETE /projects | Untrusted request must carry a valid Cognito JWT Bearer token; any request without one is rejected at the filter layer |
| JWT → Spring SecurityFilterChain | JWT is parsed and validated by Spring Security's JwtDecoder; the app never touches raw JWT bytes |
| Cognito JWKS endpoint → JwtDecoder | Spring fetches public keys from Cognito over HTTPS on first request; keys are cached (5-minute TTL) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | POST/PUT/DELETE /projects — unauthenticated caller | mitigate | SecurityFilterChain `.requestMatchers(HttpMethod.POST).authenticated()` rejects all requests without a Bearer token; enforced before controller code runs |
| T-08-02 | Spoofing | JWT issued by a different Cognito pool or issuer | mitigate | Spring Security validates the `iss` claim against `spring.security.oauth2.resourceserver.jwt.issuer-uri`; tokens from any other issuer are rejected with 401 |
| T-08-03 | Tampering | JWT signature forged or modified in transit | mitigate | Spring Security's JwtDecoder validates RS256 signature against Cognito's JWKS public key; any tampered token fails signature check |
| T-08-04 | Repudiation | Caller denies making a write request | accept | Phase 8 does not require audit logging; any authenticated Cognito user is authorised to write — RBAC/ownership tracking is out of scope (Phase 9+) |
| T-08-05 | Information Disclosure | 401 response leaks internal application structure | accept | Spring Security default 401 responses contain no stack traces or internal details; low risk for a portfolio API |
| T-08-06 | Denial of Service | Flooding write endpoints with invalid JWTs | accept | Low-value target (portfolio project); JWKS validation is CPU-light; rate limiting is out of scope for Phase 8 |
| T-08-07 | Elevation of Privilege | Expired JWT re-used after token expiry | mitigate | Spring Security's JwtDecoder validates `exp` (expiration) claim automatically; expired tokens return 401 |
| T-08-08 | Elevation of Privilege | JWT with wrong audience claim accepted | accept | `spring.security.oauth2.resourceserver.jwt.audiences` is not configured; Cognito tokens do not always include `aud` in a resource-server context; ASVS L1 does not mandate audience validation here |
| T-08-09 | Spoofing | Wrong Cognito issuer-uri format (trailing slash, App Client ID) | mitigate | issuer-uri must be exactly `https://cognito-idp.eu-west-1.amazonaws.com/eu-west-1_sDg68GJKt` — no trailing slash, no App Client ID; verified by acceptance criteria grep |
| T-08-10 | Elevation of Privilege | @Import(SecurityConfig.class) missing in test — writes appear unprotected | mitigate | All write-endpoint tests now include `@Import(SecurityConfig.class)` enforced by acceptance criteria; tests without it would silently pass for wrong reasons |
</threat_model>
